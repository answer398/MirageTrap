# Web蜜罐与态势感知系统设计文稿

## 1. 文稿目的

本文档用于说明一个毕业设计场景下的 **Web蜜罐与态势感知系统** 应实现的主要功能、业务流程、输入输出内容以及页面展示方式。本文档不涉及代码实现、开发框架、数据库设计和部署方式，重点面向希望了解该系统“能够做什么、如何运行、各功能模块之间如何配合”的读者。

---

## 2. 系统建设目标

本系统以 Web 蜜罐为核心入口，通过模拟可被访问的 Web 服务，接收外部访问请求，并对访问行为进行记录、筛选、分析和展示。系统的建设目标包括：

1. 对外提供一个可被访问的 Web 蜜罐入口，用于接收潜在攻击请求；
2. 对接收到的访问流量进行记录，保留与攻击行为相关的关键信息；
3. 对访问数据进行分析，区分正常访问与攻击访问；
4. 对已识别的攻击事件进行展示，包括请求内容、攻击来源、攻击类型和命中规则；
5. 通过态势感知页面对攻击整体情况进行统计和可视化呈现；
6. 支持对单一攻击源进行攻击过程回放，并导出相关数据包，便于分析和复盘。

简而言之，该系统要实现一个完整的基础闭环：

**接收访问 -> 记录数据 -> 识别攻击 -> 展示结果 -> 回放分析**。

---

## 3. 系统功能概述

系统主要包含以下五个功能模块：

1. **Web蜜罐采集模块**
2. **攻击识别与分析模块**
3. **攻击事件管理模块**
4. **态势感知展示模块**
5. **攻击回放与导出模块**

这五个模块共同组成系统的核心功能体系。

---

## 4. 系统总体业务流程

系统运行过程可以概括为以下步骤：

### 4.1 外部访问进入蜜罐
外部用户或攻击者向 Web 蜜罐发起请求。请求可能是普通访问，也可能是恶意访问，例如带有 SQL 注入语句、XSS 代码、路径遍历字符或恶意扫描特征的请求。

### 4.2 蜜罐记录访问内容
蜜罐接收到请求后，对请求过程进行记录，形成原始访问数据。记录内容通常包括访问来源、访问时间、请求路径、请求头、请求体等内容。

### 4.3 控制层进行分析处理
系统对蜜罐上报的数据进行筛选和判定，识别哪些请求属于攻击行为，并进一步给出攻击类型和命中规则。

### 4.4 结果进入管理与展示环节
已被识别为攻击的数据进入攻击事件列表，在管理页面中展示详细信息；同时系统将攻击数据进行统计，用于态势感知页面中的图表展示。

### 4.5 用户查看与复盘
系统使用者可以在管理页面中查看单条攻击详情，也可以在态势页面中查看整体攻击分布，还可以在攻击回放页面中按单个来源 IP 查看其攻击过程，并导出数据包。

## 5. 各功能模块设计

## 5.1 Web蜜罐采集模块

### 5.1.1 功能说明
Web 蜜罐采集模块用于对外提供一个可被访问的 Web 入口，并接收所有外部请求。该模块的主要作用不是提供真实业务功能，而是诱导外部访问行为进入系统，使系统能够采集到潜在的攻击流量和探测行为。

### 5.1.2 输入内容
该模块的输入为外部发起的 HTTP 请求，主要包括：

- 来源 IP 地址
- 访问时间
- 请求方法（如 GET、POST）
- 请求路径
- 请求头信息
- 请求参数
- 请求体内容

### 5.1.3 输出内容
该模块输出的是原始访问记录，即将接收到的请求整理为可供分析的数据。输出内容通常包括：

- 请求唯一标识
- 来源 IP
- 请求时间
- 请求方法
- 请求路径
- 请求头
- 请求体
- 原始访问内容记录

### 5.1.4 页面体现方式
该模块本身通常不直接向系统使用者展示，但它是后续所有功能的数据来源。在系统说明中，可将其体现为“数据采集入口”或“攻击入口”。

### 5.1.5 模块价值
该模块解决的问题是“如何接收到攻击”。没有这一模块，系统无法获取后续分析所需的原始数据。

---

## 5.2 攻击识别与分析模块

### 5.2.1 功能说明
攻击识别与分析模块负责对蜜罐采集到的访问数据进行筛选，区分正常访问和攻击访问，并对攻击事件进行类型判定。该模块是系统的分析核心。

### 5.2.2 输入内容
该模块的输入来自 Web 蜜罐采集模块输出的原始访问数据，主要包括：

- 来源 IP
- 请求方法
- 请求路径
- 请求头
- 请求参数
- 请求体
- 请求时间

### 5.2.3 处理内容
该模块对访问内容进行识别分析，重点判断：

1. 当前请求是否为攻击请求；
2. 若为攻击请求，属于哪一类攻击；
3. 命中了哪些攻击判定规则；
4. 是否需要进入后续展示和回放模块。

可识别的典型攻击类型包括：

- SQL 注入攻击
- 跨站脚本攻击（XSS）
- 路径遍历攻击
- 命令执行尝试
- 恶意扫描行为

### 5.2.4 输出内容
该模块输出的是分析结果，通常包括：

- 判定结果（正常/攻击）
- 攻击类型
- 命中规则名称
- 命中规则说明
- 关联的原始请求记录

### 5.2.5 页面体现方式
该模块的结果会在“攻击事件管理页面”和“攻击详情页面”中体现。系统使用者可看到：

- 某条请求是否被判定为攻击；
- 攻击属于哪一类；
- 命中了哪些规则；
- 请求体中的哪些内容具有攻击特征。

### 5.2.6 模块价值
该模块解决的问题是“如何从大量访问中识别出攻击”，决定系统能否实现蜜罐的核心分析能力。

---

## 5.3 攻击事件管理模块

### 5.3.1 功能说明
攻击事件管理模块用于集中展示已经识别出的攻击记录，使系统使用者可以查看攻击事件的基本信息和详细内容。该模块承担“查看”和“管理”两类作用。

### 5.3.2 输入内容
该模块的输入为攻击识别与分析模块输出的攻击事件数据，包括：

- 攻击来源 IP
- 攻击时间
- 请求方法
- 请求路径
- 请求体
- 攻击类型
- 命中规则

### 5.3.3 输出内容
该模块面向使用者输出的是可查看的攻击事件列表和攻击详情。

#### 攻击事件列表建议展示内容
- 事件编号
- 来源 IP
- 攻击时间
- 请求路径
- 攻击类型
- 命中规则
- 查看详情入口

#### 攻击详情建议展示内容
- 来源 IP
- 访问时间
- 请求方法
- 请求路径
- 请求头内容
- 请求参数
- 请求体全文
- 攻击类型
- 命中规则名称
- 命中规则说明

### 5.3.4 页面体现方式
该模块通常包括两个页面：

#### （1）攻击事件列表页面
用于按表格形式展示所有已识别攻击记录，便于快速浏览。使用者可以从这里查看系统近期发现了哪些攻击。

#### （2）攻击详情页面
用于查看单条攻击记录的详细内容。该页面应突出展示请求体、来源信息和命中规则，让使用者能够直观看到攻击行为本身。

### 5.3.5 模块价值
该模块解决的问题是“如何让人看见系统识别出的攻击”。它将分析结果转化为可查看、可理解的事件信息。

---

## 5.4 态势感知展示模块

### 5.4.1 功能说明
态势感知展示模块用于对攻击事件进行统计和汇总，并通过图表形式展示系统整体受到攻击的情况。该模块主要体现系统的可视化分析能力。

### 5.4.2 输入内容
该模块输入的是所有已识别攻击事件的统计数据，例如：

- 攻击次数
- 来源 IP 分布
- 攻击类型分布
- 不同时间段的攻击数量

### 5.4.3 输出内容
该模块输出的是面向整体态势的统计结果，通常包括：

- 攻击来源分布情况
- 攻击类型占比情况
- 攻击次数趋势变化
- 热点攻击来源

### 5.4.4 页面体现方式
态势感知页面应重点体现“整体情况”，而不是单条事件。建议页面包含以下几类展示内容：

#### （1）攻击总览信息
- 累计攻击次数
- 今日攻击次数
- 攻击来源 IP 数量
- 攻击类型数量

#### （2）攻击来源展示
以地图、列表或柱状图方式展示攻击来源。即使不做复杂地图，也可使用“来源 IP 排名”方式展示最活跃的攻击源。

#### （3）攻击类型展示
使用饼图、柱状图等方式展示不同攻击类型的数量分布，例如 SQL 注入、XSS、路径遍历等分别出现了多少次。

#### （4）攻击趋势展示
按时间维度展示攻击次数变化，例如按小时或按天统计攻击数量，让使用者看到攻击活动的变化趋势。

### 5.4.5 模块价值
该模块解决的问题是“如何从整体上理解系统受到的攻击情况”。相比攻击事件管理模块主要看单条数据，态势模块更强调全局统计和直观展示。

---

## 5.5 攻击回放与导出模块

### 5.5.1 功能说明
攻击回放与导出模块用于对某一攻击来源进行集中展示，帮助使用者分析该来源在一段时间内发起的攻击过程。同时，该模块支持导出相关数据包，以便后续复盘或展示。

### 5.5.2 输入内容
该模块输入的是已经被系统识别出的攻击记录，并以某个来源 IP 为主线进行聚合。输入通常包括：

- 来源 IP
- 与该 IP 相关的所有攻击记录
- 各条记录的访问时间
- 各条记录的请求内容
- 原始数据包信息

### 5.5.3 输出内容
该模块输出的是按单一攻击源整理后的攻击过程，包括：

- 某一来源 IP 的攻击记录集合
- 按时间顺序排列的攻击过程
- 对应请求内容和攻击类型
- 可导出的 pcap 数据包

### 5.5.4 页面体现方式
该模块通常以“攻击回放页面”的形式体现，建议包括：

#### （1）IP 选择区域
允许使用者选择一个来源 IP 进行查看。

#### （2）时间线展示区域
按时间顺序展示该 IP 发起的访问记录，例如：
- 第一次访问时间
- 第二次访问时间
- 每次请求的路径和请求体
- 每次命中的攻击类型

#### （3）数据导出区域
提供导出操作，用于导出与该攻击源相关的数据包，便于后续复盘或作为展示材料。

### 5.5.5 模块价值
该模块解决的问题是“如何对单个攻击源进行更细致的复盘分析”。相较于态势页面的整体展示，该模块更加聚焦具体攻击来源和攻击过程。

## 6. 系统输入与输出说明

为了便于不熟悉系统的人理解，本节从整体上说明系统的输入和输出。

## 6.1 系统输入
系统的输入主要来自外部访问行为，也就是进入 Web 蜜罐的请求。输入数据包括：

- 访问来源 IP
- 访问时间
- 请求方法
- 请求路径
- 请求头信息
- 请求参数
- 请求体内容

这些内容是系统判断攻击行为的基础。

## 6.2 系统输出
系统的输出主要分为三类：

### （1）攻击识别结果输出
即系统对输入请求的分析结果，包括：

- 是否为攻击
- 攻击类型
- 命中规则

### （2）页面展示输出
即系统前端页面向使用者展示的内容，包括：

- 攻击事件列表
- 攻击详情信息
- 攻击来源统计
- 攻击类型分布
- 攻击趋势变化
- 单一 IP 攻击过程

### （3）数据导出输出
即供分析和留存使用的导出内容，包括：

- 单一攻击源的 pcap 数据包
- 攻击记录整理结果

## 8. 系统使用者视角说明

从系统使用者的角度看，本系统可以回答以下问题：

1. **系统是否收到了外部攻击？**
   - 可以，通过攻击事件列表和态势页面可以直接看到。

2. **攻击来自哪里？**
   - 可以，通过来源 IP 和来源分布统计可以看到。

3. **攻击属于什么类型？**
   - 可以，通过攻击类型和命中规则可以判断。

4. **攻击请求具体是什么内容？**
   - 可以，通过攻击详情页面中的请求头、参数和请求体可以查看。

5. **某一个攻击者的行为过程是什么？**
   - 可以，通过攻击回放页面按单个 IP 查看。

6. **是否支持导出用于进一步分析的数据？**
   - 可以，系统支持导出 pcap 数据包。

因此，本系统不仅能够“接收到攻击”，还能够“识别攻击、展示攻击、分析攻击”。

## 9. 功能边界说明

为了突出毕业设计重点，系统功能范围主要控制在基础 Web 蜜罐和态势感知功能之内，不扩展到更复杂的企业级安全联动场景。本文档所描述的系统功能边界包括：

### 本系统重点实现
- Web 请求采集
- 攻击识别
- 攻击事件展示
- 态势统计展示
- 单 IP 攻击回放
- pcap 导出

### 本系统暂不重点涉及
- 外部告警联动
- 自动封禁处置
- 多协议蜜罐扩展
- 大规模分布式部署
- 复杂权限管理
- 高级威胁情报联动

这样的边界划分符合毕业设计“功能完整、结构清晰、实现可控”的要求。

## 10. 总结

本系统是一个面向毕业设计场景的基础 Web 蜜罐与态势感知系统。系统通过 Web 蜜罐接收外部访问请求，记录请求过程和请求内容，并对访问流量进行分析，识别其中的攻击行为。对于识别出的攻击，系统能够在管理页面中展示详细信息，并在态势感知页面中展示整体攻击来源、攻击类型和攻击趋势。同时，系统支持按单一来源 IP 对攻击过程进行回放，并导出 pcap 数据包。

从功能设计角度看，该系统实现了一个较为完整的基础闭环，即：

**采集、识别、展示、统计、回放。**

对于希望了解系统功能的人来说，这一系统能够直观体现：

- 它从哪里获取数据；
- 它如何判断攻击；
- 它向使用者展示什么内容；
- 它如何帮助使用者理解攻击过程。

因此，该设计能够较好满足毕业设计中对 Web 蜜罐与态势感知系统的功能性要求。
